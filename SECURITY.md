# Security Policy

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting form from this repository's
**Security** tab. Include the affected component and version, reproduction
steps, impact, and any suggested mitigation.

Please allow the maintainers time to investigate before public disclosure. We
will acknowledge the report, coordinate remediation, and credit reporters who
want attribution.

Never include real API keys, access tokens, audio, or full transcripts in a
report.
